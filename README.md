# Self-Consistency as a Predictor of Accuracy — De-identified benchmark results

This repository releases the **de-identified benchmark results** underlying the study
of self-consistency (agreement across stochastic samples) as a confidence/abstention/
routing signal for large language models on hard reasoning benchmarks (GPQA Diamond and
AIME). It contains **only derived, de-identified per-run rows and answer-count
distributions** — enough to reproduce the paper's clustered analyses.

- **265,000** samples · **53** anonymized runners · **394** unique cases · **5,300** case-result rows
- Benchmarks: **GPQA Diamond** (4-option MCQ) and **AIME** (integer answers)
- Models: `gpt-4.1-nano`, `gpt-4.1-mini`, `gpt-4.1` (temperature 1.0, K=50)

> **Not included (by design):** GPQA Diamond question text is **not** redistributed
> (consistent with its access terms); only case IDs and answer-distribution metrics are
> released. No student/runner personal data is present — all identifiers were removed and
> replaced with anonymous indices (`S001`–`S053`).

## Contents

```
data/
├── case_results_deid.parquet        # 5,300 × 24 — canonical tidy per-run rows (also as .csv)

```

## Data dictionary — `case_results_deid.parquet`

| column | type | meaning |
|---|---|---|
| `student_id` | str | anonymized runner index (`S001`–`S053`) |
| `axis` | str | controlled comparison: **A** model tier, **B** prompt strategy, **C** mid vs. frontier |
| `condition` | str | `a` or `b` within the axis |
| `axis_ok`, `meta_ok` | bool | validation flags (submitted axis/meta match the assignment) |
| `case_id` | str | `gpqa_diamond_NNN` or `aime_NNN` (no question text) |
| `benchmark` | str | `gpqa_diamond` or `aime` |
| `model` | str | `gpt-4.1-nano` \| `gpt-4.1-mini` \| `gpt-4.1` |
| `prompt` | str | `zero_shot` \| `chain_of_thought` |
| `K` | int | samples drawn per case (50) |
| `majority_answer` | str | modal answer over K samples (letter, integer-as-string, or `_UNPARSEABLE_`) |
| `ground_truth` | str | correct answer (same form as `majority_answer`) |
| `subject` | str | GPQA subject (Physics/Chemistry/Biology) or `None` for AIME |
| `year` | int | AIME problem year (or `None`) |
| `n_distinct_answers` | int | number of distinct answers observed over K |
| `n_unparseable` | int | count of `_UNPARSEABLE_` samples |
| **`C`** | float | **self-consistency** = n_majority / K |
| **`A`** | float | **sample accuracy** = n_correct / K |
| **`majority_is_correct`** | bool | **M** — deployment label, `majority_answer == ground_truth` |
| `R_0`, `R_1`, `R_2` | float | accuracy / Wilson-interval helper columns per the project spec (`R_0 == A`) |
| `R_wilson`, `R_wilson_recomputed` | float | submitted vs. recomputed Wilson-score bound (max diff ≈ 0.07) |

`answer_distributions_deid.parquet` carries the same keys plus **`answer_counts`** (a JSON
map `answer → count` over the K samples) and `total` (= K).

The three primary quantities in the paper are **`C`** (self-consistency), **`A`** (sample
accuracy), and **`majority_is_correct`** (the deployment label M).

## Quick start

```python
import pandas as pd
df = pd.read_parquet("data/case_results_deid.parquet")
# Spearman correlation of self-consistency with majority-correctness, per cell:
print(df.groupby(["axis", "condition", "benchmark"])[["C", "majority_is_correct"]]
        .corr(method="spearman").unstack().iloc[:, 1])
```

## Provenance & ethics

This is a **secondary, de-identified meta-analysis** of model-output data generated by 53
runners completing a graduate-course assignment. All personal identifiers were removed
prior to analysis and replaced with anonymous indices; no individual is identifiable. GPQA
Diamond question text is not redistributed. No API calls were re-run — every metric derives
from the submitted run-level outputs.
